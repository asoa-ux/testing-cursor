Navigation Actions

From List View

Click tree node → Table shows its children
Click table name → Open that item's Detail View
View Details btn → Open selected node's Detail View
Breadcrumb → Go to that node's List View

From Detail View

← button → Back to List View
Esc key → Back to List View
Prev / Next → Navigate to sibling Detail
← → keys → Navigate to sibling Detail
Click sibling → Open that sibling's Detail
Breadcrumb → Go to that node's List View

Key Design Decisions

1. Tree is hidden in Detail View to avoid conflicting selection states
2. Sibling list provides contextual navigation (only relevant items)
3. Breadcrumb always goes to List View (not Detail) for consistency
4. Multiple ways to go back (button, key, link) for discoverability
5. "Back" always returns to where you came from, not necessarily the parent
