Design Rationale

  • No pre-highlighting: The default view does not highlight differences because users may have various intents — reviewing completeness, auditing data, browsing attributes — not just comparing. Pre-highlighting assumes comparison intent and adds visual noise for other tasks.
  • Intent-driven highlighting: Visual emphasis (gray background, left bar) only appears when the user explicitly enables "Show differences only." This ties visual treatment to stated user intent.

Interaction Notes

  • Default view: Clean, neutral table with no pre-highlighting.
  • Toggle behavior: "Show differences only" filters to rows where values differ. Highlighting only appears in filtered view.
  • Difference detection: An attribute is marked as "different" if its value varies in any revision column.
  • Row-level indication: When filtered, differences are shown at the row level (entire row), not per individual cell.
  • Visual indicators: Filtered difference rows use a light gray background and a left-side vertical bar accent.
  • Truncated content: Long text values represented as gray placeholder blocks to indicate truncation.
  • Column flexibility: Table structure supports 2+ revision columns for comparison.
