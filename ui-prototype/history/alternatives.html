Alternative 1: Timeline with Collapsible Value Cards

A vertical timeline where each change is a card. Unchanged revision spans are collapsed but visible. Works well for long text fields as content wraps naturally.

Pros

  • Clear visual flow of time
  • Long descriptions wrap naturally
  • Unchanged spans are acknowledged but not cluttering
  • Expandable for details when needed

Cons

  • Takes more vertical space than a table
  • Comparing non-adjacent versions requires scrolling

Alternative 2: Diff-First View

Shows what changed using inline diff highlighting (like Git). For long descriptions where only a few words changed, this is far more useful than showing full before/after values.

Pros

  • Immediately see what changed, not just that it changed
  • Very compact for large text fields
  • Familiar pattern from code review tools
  • Option to show full value when needed

Cons

  • Diff algorithm can be confusing for major rewrites
  • More complex to implement

Alternative 3: Expandable Table with Smart Grouping

Keeps the familiar table layout but makes it smarter: rows expand to show full values, and unchanged revision ranges are collapsed into single rows.

Pros

  • Familiar table layout for data-oriented users
  • Expandable rows handle long text
  • Filtering by editor
  • Clear grouping of unchanged revisions

Cons

  • Expansion pattern may not be obvious
  • Can still feel cramped for very long text

Alternative 4: Two-Panel Comparison View

Side-by-side comparison of any two versions. Users choose which versions to compare using dropdowns. Excellent for long text where you need to see full context.

Pros

  • Excellent for long text comparison
  • User controls what to compare
  • Clear when values are identical
  • Shows full context, not just snippets

Cons

  • Requires wider popover
  • Less useful for attributes with few changes
  • Doesn't show full timeline at once

Alternative 5: Activity Feed Style

A conversational, activity-feed approach similar to Git commits or Slack updates. More human-readable with natural language descriptions of changes.

Pros

  • Modern, familiar pattern
  • Natural language is easy to scan
  • Visual distinction between change types
  • Feels less "database-y"

Cons

  • Takes more vertical space
  • Less precise than a table for audit purposes
